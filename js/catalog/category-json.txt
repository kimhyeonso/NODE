category.json 설명

이 파일은 PRODUCT와 ACCESSORY가 함께 사용하는 카탈로그 데이터 원본입니다.
상품 카드 HTML을 넣어두는 파일이 아니라, 카드와 상세페이지에 들어갈 데이터를 넣어두는 파일입니다.

역할 구분

1. category.json
   - 상품명, 브랜드, 가격, 원가, 이미지, 설명, 카테고리, 필터값, 스펙을 보관합니다.
   - studio, live, cable, stand 데이터를 한 파일에서 관리합니다.

2. product.js
   - category.json에서 studio/live/cable/stand 데이터를 읽습니다.
   - product-main.html, product-studio.html, product-live.html의 상품 카드를 자동 생성합니다.
   - accessory-main.html, accessory-cable.html, accessory-stand.html의 액세서리 카드도 자동 생성합니다.

3. product-detail.js
   - category.json에서 상품 상세 정보와 스펙을 읽습니다.
   - 제품과 액세서리 모두 같은 product-detail.html을 사용합니다.

중요한 필드

- fallbackId
  상세페이지에서 잘못된 id가 들어왔을 때 기본으로 보여줄 상품 id입니다.

- sharedDetailImages
  상세페이지에서 공통으로 사용하는 상세 이미지입니다.

- products
  각 상품의 기본 데이터입니다.
  name, brand, price, originalPrice, category, filters, images, description을 넣습니다.

- category
  목록 페이지에서 어떤 상품을 보여줄지 판단할 때 씁니다.
  사용 값: studio, live, cable, stand

- filters
  필터 체크박스와 상품을 연결하는 값입니다.
  예: ["near-field", "2-way", "flat", "under-50"]
  예: ["xlr", "microphone", "1m"]

- specifications
  상세페이지와 비교 페이지의 스펙 표에서 쓰는 데이터입니다.

정리하면:
category.json = 제품과 액세서리의 통합 데이터
product.js = studio/live/cable/stand 카드 생성
HTML 파일 = 카드가 들어갈 자리
